# 首页重规划与线框图

更新时间：2026-03-27

## 目标

把首页从“概念展示页”改成“玩家落地页”：

- 新用户 10 秒内知道这是什么
- 已有钱包用户 30 秒内进入扫描
- 玩家能快速理解 `扫描 -> 进度 -> 可领取 -> 分享身份`
- 首页减少重复解释，把真实操作入口前置

## 当前首页的核心问题

### 1. 第一屏信息太满，但行动路径不够直

- 现在 Hero 同时塞了概念、信号卡、预览终端、二级导航、钱包入口
- 用户能看出“很酷”，但不一定能立刻知道“我下一步该点哪”

### 2. 真实主流程被压得太后

- 真正最关键的 `Live Chronicle` 在页面较后位置
- 首页前半段讲了很多系统能力，后半段才进入玩家自己的数据

### 3. 价值表达有点散

- `Combat Score`
- `Warrior Card`
- `Protocol`
- `Proof Classes`

这些都重要，但首页不该平铺同权，得有主次。

### 4. 对新手不够友好

- 玩家最关心的是：
  - 我连钱包后会发生什么
  - 系统认哪些行为
  - 为什么我现在不能 Claim
- 这些答案现在分散在多个区块里

## 新首页信息架构

建议把首页改成 6 个主模块，按这个顺序排：

1. Hero 首屏
2. How It Works 三步流程
3. Live Chronicle 快速入口
4. Achievement Preview 成就与奖励预览
5. Proof & Trust 可信机制说明
6. Warrior Card 分享身份

## 模块说明

### 1. Hero 首屏

职责：

- 一句话讲清产品
- 给出唯一主动作
- 给出一个次动作

建议文案方向：

- 主标题：`把你在 Frontier 的行为，变成可验证的链上资历`
- 副标题：`连接钱包后，系统会扫描你的跃迁、锚定、击杀记录，计算成就进度，并在可达标时开放勋章领取。`
- 主按钮：`Connect Wallet & Scan`
- 次按钮：`View Sample Warrior Card`

首屏右侧不要再放复杂大终端，改成“结果预览卡”：

- 扫描状态
- 可领取勋章数
- 当前战力分
- 最近一次识别到的行为

这样用户第一眼看到的是“结果”，不是“系统自我介绍”。

### 2. How It Works 三步流程

职责：

- 把系统逻辑讲简单
- 解决“这玩意到底怎么运作”的疑虑

建议三步：

1. `Connect Wallet`
2. `Scan Frontier Activity`
3. `Claim Verified Medals`

每步只保留一句解释，别写论文。

### 3. Live Chronicle 快速入口

职责：

- 直接承接首屏 CTA
- 把真实扫描模块前置到首页中上部

这个区块应该尽早出现，不要再压到页面底部。

区块内部建议分成两列：

- 左侧：钱包连接、网络状态、环境说明
- 右侧：ChronicleDashboard

环境说明只保留 3 条最关键提示：

- 默认按 `testnet` 校验
- 没有 `EVE_EYES_API_KEY` 会进入 preview mode
- 当前网络没有 package ID 时只能看进度不能 Claim

### 4. Achievement Preview 成就与奖励预览

职责：

- 告诉玩家“你能获得什么”
- 把成就体系讲得更像目标，不像配置表

建议把当前 `Proof Classes` 和 `ScoreShowcase` 收束成一个区块：

- 左边：3 个代表性成就卡
  - Galactic Courier
  - Void Pioneer
  - Bloodlust Butcher
- 右边：战力分规则摘要
  - claimed = full score
  - verified but unclaimed = partial score
  - full set bonus

重点是“可理解”，不是把全部规则一次性砸脸上。

### 5. Proof & Trust 可信机制说明

职责：

- 回答“凭什么信你”
- 给想看底层逻辑的用户一个明确解释

建议内容：

- 数据来源：Eve Eyes
- 判定对象：跃迁 / 锚定 / 击杀
- 上链结果：Sui soulbound medals
- Claim 限制：只有满足门槛才可领取

这个模块可以保留一点终端感，但不要抢首屏戏份。

### 6. Warrior Card 分享身份

职责：

- 给用户一个“完成扫描后的可传播结果”
- 服务招募、社群身份展示、战绩背书

保留现有 `WarriorCallout` 的方向，但建议位置后置。

因为分享卡不是首页第一任务，它是结果页价值，不是进入站点的第一驱动力。

## 推荐导航结构

顶部导航建议精简成 4 项：

- `How It Works`
- `Achievements`
- `Live Chronicle`
- `Warrior Card`

把现在首页内部那组 `Combat Score / Warrior Card / Protocol / Live Chronicle` 的首屏子导航砍掉。

原因很简单：

- 顶部 Header 已经有导航
- 首屏里再来一套，重复且抢注意力
- Hero 的按钮应该只服务转化，不该兼任站内目录

## 内容优先级建议

必须前置：

- Connect Wallet
- Live Chronicle
- 扫描结果预期
- 成就价值

可以后置：

- 详细分数公式
- 复杂协议解释
- 大段系统叙事

该删就删：

- 重复出现的“系统能力说明”
- 首屏里过多的终端装饰性模块
- 同层级竞争注意力的多个入口

## 视觉方向建议

视觉上继续沿用当前项目已经建立的基调：

- `martian red` 作为主强调色
- `crude / steel / dark` 作为底色
- 保留工业终端感和扫描感

但要做两个收缩：

- 首屏更像“作战入口”，少一点“概念展板”
- 中下部再释放系统解释和终端细节

## 实施顺序建议

如果下一步要正式改代码，建议按这个顺序做：

1. 重排 `packages/nextjs/src/app/page.tsx` 的 section 顺序
2. 把首屏 Hero 改成“结果导向”的双栏结构
3. 把 `ChronicleDashboard` 上移到首页中上部
4. 合并 `ScoreShowcase` 与 `Proof Classes` 的表达
5. 把 `WarriorCallout` 保留为后置价值区块
6. 最后再统一细节文案、CTA 和锚点命名

## 结论

这版首页不该再继续堆“我们系统很牛逼”的自嗨展示了，核心是把玩家往真实扫描和真实结果上推。

一句话总结新的首页逻辑：

`先告诉玩家能得到什么，再让他立刻开始扫描，最后再解释为什么这套系统可信。`
