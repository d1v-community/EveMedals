# EVE Medals 项目简介

更新时间：2026-03-31

## 项目一句话

`EVE Medals` 把 `EVE Frontier` 玩家真实发生过的行为，变成可验证的勋章、Warrior 档案和可分享卡片。  
它通过索引玩家在游戏中的关键行为，生成可验证的勋章状态、链上荣誉凭证和可分享的 Warrior / Medal 卡片，把玩家战绩转化为可展示、可传播的身份资产。

## 这个项目现在在做什么

当前项目的主线是：

`玩家行为索引 -> 成就判断 -> 勋章领取 -> Warrior 展示 -> 社交分享`

具体来说，系统会：

1. 扫描玩家在 `EVE Frontier` 中的部分可索引行为
2. 将这些行为映射为勋章进度与成就状态
3. 在满足条件后，支持玩家将勋章领取到 Sui 钱包
4. 在 Warrior 页面展示玩家的勋章、分数、身份信息
5. 自动生成可分享的战绩卡片与勋章卡片

## 当前已经具备的能力

### 1. 成就扫描与验证

- 基于 `Eve Eyes` 扫描玩家行为
- 生成 `Chronicle snapshot`
- 支持 8 枚勋章的进度计算和状态展示
- 区分 `locked / verified / claimable / claimed`

### 2. 链上勋章能力

- 使用 Sui Move 合约承载勋章对象
- 支持勋章模板、领取状态和链上回查
- 玩家可在满足条件后发起领取
- Warrior 页面可读取钱包已持有勋章

### 3. Warrior 展示能力

- 提供独立 Warrior 页面
- 展示钱包地址、角色信息、勋章概览、分数与 rank
- 支持将 Warrior 页面作为分享落地页

### 4. 社交分享能力

- 已有 `Warrior Profile` 总卡分享
- 已有单勋章 `Medal Share` 分享卡
- 支持动态图片生成
- 支持复制链接、下载卡片
- 支持 `X / Telegram / Discord` 分发入口
- 已支持二维码

## 当前更准确的产品定位

如果用一句更贴近当前代码现实的话来描述，这个项目现在更像：

> 一个基于 `Eve Eyes + Sui` 的 EVE Frontier 成就验证与分享系统。

它已经具备了：

- 可验证的成就状态
- 链上荣誉承载
- Warrior 身份展示
- 社交传播素材

但它现在还不能完全算作一个成熟的“社交增长引擎”，更适合定义为：

> 一个 `working achievement verification and sharing product`。

## 当前与“社交增长引擎”目标之间的差距

### 1. 还没完全形成增长闭环

目前虽然已经有分享卡、分享链接和二维码，但还缺少真正的增长系统能力，例如：

- referral / 邀请参数
- campaign 跟踪
- 分享来源归因
- 访问到连接钱包的转化统计
- 分享到领取的漏斗分析

所以当前更适合定义为：

> 已具备传播入口的成就产品

而不是：

> 已完成裂变增长闭环的增长引擎

### 2. 当前并非纯链上驱动

这个项目不是单纯把链上数据包装成卡片，而是：

- 链下通过 `Eve Eyes` 索引玩家行为
- 链上通过 Sui 记录勋章持有结果

所以更准确的描述应该是：

> 把行为验证结果和链上持有状态组合成可传播资产

### 3. Testnet 真实合约对接尚未完成

当前项目还没有完全对接最终可信的 `testnet` 合约环境。  
这意味着：

- Demo 与开发态能力已经具备
- 但正式对外讲“完整 testnet 验证闭环已跑通”还不够稳

## 当前需要特别说明的现实情况

为了避免误解，下面这些点需要明确：

### 已有的

- 分享卡不是没有，已经有
- 二维码不是没有，已经有
- Warrior 页面不是静态页，已经和勋章/快照逻辑打通

### 还不够完整的

- 二维码当前主要回到站内验证页，不是直接跳链上浏览器
- Discord 目前更接近“复制链接后发送”，不是原生一键分享
- 增长归因与裂变机制还没有真正补齐
- 真正可对外背书的 testnet 合约对接还差最后一步

## 当前最稳妥的对外说法

建议对外使用下面这版：

> EVE Medals 是一个面向 EVE Frontier 的成就验证与社交传播产品。  
> 我们将玩家的 Frontier 行为索引结果与 Sui 链上勋章结合，生成 Warrior 档案和可分享卡片，把游戏战绩转化为可验证、可展示、可传播的荣誉资产。

如果要保留“增长”方向，可以用更稳一点的版本：

> EVE Medals 正在从成就验证产品演进为社交传播与增长基础设施。  
> 当前阶段已经完成勋章验证、链上领取、Warrior 展示和分享卡片能力，下一步将继续补齐 referral、归因和裂变增长机制。

## 总结

当前这个项目已经不是一个空壳概念，而是一个真实在跑的：

- 成就验证系统
- 勋章领取系统
- Warrior 展示系统
- 分享传播系统

但它距离“成熟的社交增长引擎”还差几步，主要差在：

- testnet 真合约对接
- 增长归因
- referral / campaign
- 分享转化闭环

所以现阶段最准确的判断是：

> 它已经是一个完整的成就验证与分享产品雏形，但还不是最终形态的增长引擎。
